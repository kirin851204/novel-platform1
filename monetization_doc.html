05 ウォレット構造（2テーブル）
💰 購入ポイント
paid_points
✓課金して取得
✓すべての機能に使える
✓作家への送金・投げ銭に使える
✗現金化はできない
🎯 行動ポイント
free_points
✓行動報酬で取得
✓AI対話・有料話の購入に使える
✗作家への送金・投げ銭には使えない
✗現金化はできない
📌ポイントは 行動ポイント → 購入ポイント の順で使われます。購入ポイントをあとに残すことで、課金した価値を感じやすくします。
wallets ユーザーごとに1件の情報を持つテーブル
項目名
データ型
内容
id
uuid / PK
このウォレット情報を識別するためのID
user_id
uuid / FK
どのユーザーのウォレットかを示すID（users.id を参照）
paid_points
int
購入したポイントの残高
free_points
int
行動によって獲得したポイントの残高
updated_at
timestamptz
この情報が最後に更新された日時
🔒 ログイン中の本人だけが自分のデータを見られる設定（RLS）
point_transactions ポイントの増減履歴を記録するテーブル
項目名
データ型
内容
id
uuid / PK
この取引履歴を識別するためのID
user_id
uuid / FK
どのユーザーの取引かを示すID（users.id を参照）
type
text
取引の種類（例：行動で獲得、購入で獲得、消費、送金）
point_kind
text
どちらのポイントか（購入ポイント / 行動ポイント）
amount
int
ポイントの増減量（プラスは獲得、マイナスは消費）
reason
text
増減した理由（例：レビュー、投稿、AI対話など）
ref_id
uuid
関連する作品やレビューなどのID
created_at
timestamptz
この取引が記録された日時
🔒 ログイン中の本人だけが自分の履歴を見られる設定（RLS）
wallets 1 ─── ∞ point_transactions 1つのウォレットに対して、複数の取引履歴がひもづきます
